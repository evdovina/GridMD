	Оформление документации проекта с помощью Doxygen

1. Документация по ivutils и GridMD ведется отдельно, из
   GridMD возможны ссылки на ivutils. Конфигурационные файлы находятся в ivutils/doxygen/Doxyfile и 
   gridmd/doc/doxygen2/Doxyfile

2. Примером правильного оформления документации является файл ivutils/include/Plane_3.h


3. Для того, чтобы компилировались встроенные в документацию формулы, помимо doxygen нужно
   установить Miktex и ghostscript. Путь к gswin32c.exe должен быть прописан в переменной окружения PATH
   (Компьютер->Свойства->Дополнительные параметры системы-> Переменные среды)

4. Английская и русская документация содержится в коде одновременно и переключается тегами
   \en и \ru. Эти теги являются макроопределениями (ALIASES) doxygenа.
   en=\endif \if EN
   ru=\n\endif \if RU
   По правилам doxygen секции EN и RU должны закрываться, т.е. иметь вид \if RU ... \endif
   МЫ ИГНОРИРУЕМ ЭТО ПРАВИЛО, и можем писать для краткости \en и \ru без закрытий (сооответствий \if и \endif), 
   doxygen при этом выдает предупреждения, но генерирует HTML нормально.
   

4. По умолчанию в файле Doxyfile настроено воспроизведение в HTML английской и русской документации одновременно,
   с макетом сайта (навигация, заголовки) на русском.
   
   Для переключения на полностью английский вариант нужно в Doxywizard:
   - убрать RU из Expert->Build->ENABLED_SECTIONS
   - поставить Expert->Project->OUTPUT_LANGUAGE  English

   Для переключения на полностью русский вариант нужно в Doxywizard:
   - убрать EN из Expert->Build->ENABLED_SECTIONS


5. Чтобы вывод был красивым, а код в едином стиле, надо соблюдать следующие правила:
   5.1 Правила отступов для кода С++:
       
       отступ блока С++ -- 2 пробела, символ табуляции КАТЕГОРИЧЕСКИ НЕ ПРИМЕНЯЕТСЯ 
       (поставить в VS: Tools->Options->TextEditor->AllLanguages->Tabs (Tabs Size 2, Indent Size 2, Insert Spaces)  

       стиль блоков C++ : открывающаяся скобка на той же строке с определением, закрывающаяся -- выравнена с началом 
       блока по колонке, например
       for(){
         if(a)
           function(); 
          
         ...
       }
       do{
         ...
         for(){
           ...
         }
       }while();


       смещения СТРОГО соответствуют уровню вложенности блоков, произвольные отступы для красоты не допускаются

   5.2 для комментариев:
       английский текст блока комментария сначала, русский потом (см. \n в определении \ru)
       /// \en  Brief comment. 
       ///      Detailed comment
       ///      \param[in] a
       ///      \return value text
       /// \ru  Кратикий комментарий. 
       ///      Детальное описание
       ///      \param[in] a
       ///      \return value текст
       int function(){}
  
   5.3 после первого предложения описания функции ставится точка, оно автоматически попадает в краткое описание
       После описаний параметров и возвращаемых значений точки не ставятся.
   

6. Полезные теги doxygen (можно дополнять)
       \file filename   обязательно для включения файла в документацию 
       \a               выделение курсивом (ссылка на аргумент функции)
       \f$...\f$        inline-формула  LaТЕХа
       \param[in|out]   входной|выходной параметр функции
       \n               перевод строки
       \ref link_name "Highlited text" ссылка на секцию link_name, которая будет выглядеть как Highlited text в HTML
